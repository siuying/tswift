# EventKit

EventKit is Apple's calendar and reminders framework. tswift models it as an
**in-memory event store**: `EKEventStore` owns calendars, events, and reminders
as ordinary runtime values, so a program can create, mutate, save, and remove
them without a real CalendarStore database or system permission prompts.

**Current status: the full in-scope EventKit surface is shipped.** Every scoped
member of `EKEventStore`, `EKEvent`, `EKReminder`, `EKCalendar`, `EKCalendarItem`,
`EKAlarm`, `EKRecurrenceRule`, `EKParticipant`, and all EventKit enums is
implemented and reachable, and all but the enum `init(rawValue:)` keys are
exercised by a tagged golden fixture.

The coverage reference marks the 16 enum `init(rawValue:)` members **partial**:
they are implemented and reachable, but the shared builtin-enum core does not yet
round-trip an explicit per-case raw value through a fixture, so the website
reserves ✅ for fixture-verified APIs. See
`docs/swift-runtime/blocked-features.md`.

## What's wired

- **Event store** — `EKEventStore` initialisers, `authorizationStatus(for:)`,
  `requestFullAccess*`, `save`/`remove`/`commit`/`reset`, `calendars(for:)`,
  `sources`, and identifier lookups all run against an in-memory store. Identity
  is assigned on save, matching `EKObject` semantics.
- **Calendar items** — `EKEvent` and `EKReminder` are constructed with
  `EKEvent(eventStore:)` / `EKReminder(eventStore:)`. The shared `EKCalendarItem`
  and `EKObject` surface (alarms, recurrence rules, notes, change tracking) is
  folded onto both concrete receivers.
- **Calendars & sources** — `EKCalendar`, `EKSource`, and their type/colour/
  capability properties.
- **Alarms, recurrence, participants** — `EKAlarm`, `EKStructuredLocation`,
  `EKRecurrenceRule`, `EKRecurrenceEnd`, `EKRecurrenceDayOfWeek`, and read-only
  `EKParticipant` attendee/organizer values.
- **Enums** — every EventKit enum (`EKAuthorizationStatus`, `EKEntityType`,
  `EKSpan`, `EKEventStatus`, `EKParticipant*`, `EKWeekday`, …) including the
  deprecated aliases.

## Scope

EventKit is an Apple-platform-only framework with no web/wasm equivalent, so the
runtime targets the iOS-simulator SDK surface. Objective-C-only and
system-integration APIs that cannot run headless — predicate-based fetch
queries, `NSPredicate` event enumeration, UI presentation, and cross-process
change notifications — are counted separately as out of scope rather than
implying support. The store is a faithful in-memory model, not a bridge to the
host's real calendar database.
