# Foundation

Foundation is Apple's "base layer" framework — it provides date/time handling,
URL parsing, JSON encoding, data buffers, and more. tswift measures Foundation
coverage through generated `.swiftinterface` inventories and per-type runtime
registries.

**Legend:** ✅ Done · 🟡 Partial · ⬜ Todo

---

## Overall coverage

<CoverageBar label="Foundation" done={56} total={519} />

Member-level coverage from `tools/framework-inventory/coverage.py --framework
foundation`: **56 / 519** public members implemented across the in-scope types
(10.8%). The bar tracks members, not whole types — most implemented types are
still partial.

---

## How coverage is measured

Foundation coverage is tracked via the **framework inventory** tool
(`tools/framework-inventory/`). It generates a snapshot of every public type and
member in the real Apple Foundation `.swiftinterface`, then compares it against
the runtime's native registry (`frameworks/foundation/registered_keys.txt`). Run
`python3 tools/framework-inventory/coverage.py --framework foundation` for the
live roll-up. See `docs/swift-runtime/stdlib-inventory.md`.

### Out of scope

Platform- and device-specific APIs are intentionally excluded (see
`frameworks/foundation/scope.toml`): `FileManager`, `FileHandle`, `URLSession`,
`URLRequest`, `NSObject`, `Bundle`, and friends.

## Roadmap

The next coverage push (daily-use weighted) is tracked in issues
[#214–#221](https://github.com/siuying/tswift/issues?q=is%3Aissue+label%3Aready-for-agent+foundation),
in this order:

1. `Date` / `DateComponents` / `Calendar` / formatters (F3) — date math & ISO strings
2. `Decimal` / `NumberFormatter` (F4) — exact money math & number formatting
3. `Measurement` / `Unit` (F4) — typed quantities & conversion
4. Fill remaining `Data` / `UUID` / `URL` member gaps (F1/F2)
