# Standard Library

The Swift standard library provides the runtime behaviour for all built-in
types and protocols. tswift implements these entirely in Rust inside `tswift-std`.

The frontend provides type _shapes_; `tswift-std` provides the _behaviour_.

**Legend:** ✅ Done · 🟡 Partial · ⬜ Todo · 🛡️ Hardened — edge cases ported
from Apple's Swift test suite (`swift/test`) and ground-truthed against the
real Swift 6.3 toolchain.

Member counts come from `python3 tools/stdlib-inventory/coverage.py` —
a member is **verified** only when it is registered in the live runtime *and*
exercised by a passing golden fixture.

## Known divergences

Documented honestly rather than papered over:

- **Optional-in-collection printing** — `[Optional("x"), nil]` prints as
  `["x", nil]`; the value model has no Optional wrapper type.
- **Fitzpatrick skin-tone modifiers** — grapheme segmenter treats
  emoji + modifier as two Characters (ZWJ families work correctly).
- `Optional.take()` / `.debugDescription` on present optionals need
  declared-type-aware dispatch (deferred).

2. **`Collection` conformances for custom types** — custom types in `for-in`,
   `count`, index subscripting.
3. **`AsyncSequence`** — `for await` over custom types.
