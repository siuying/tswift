# Standard Library

The Swift standard library provides the runtime behaviour for all built-in
types and protocols. tswift implements these entirely in Rust inside `tswift-std`.
The frontend provides type _shapes_; `tswift-std` provides the _behaviour_.

**Legend:** ✅ Implemented · 🟡 Partial · ⬜ Missing · 🚫 Out of scope.
Member counts are generated by `python3 tools/framework-inventory/coverage.py
--framework stdlib --emit-json` from the live runtime registry — nothing on
this page is hand-typed. Use the search box and status chips below to explore
the full member list.

## Known divergences

Documented honestly rather than papered over:

- **Fitzpatrick skin-tone modifiers** — the grapheme segmenter treats
  emoji + modifier as two `Character`s (ZWJ families work correctly).
- **`Unsafe*Pointer` family, `withUnsafe*`, `transcode`, va-list** — not
  honestly implementable in a tree-walking interpreter; intentionally
  excluded rather than faked.

## What's next

1. **Remaining `String`/`Substring` members** — encoding, span, and mutable
   view APIs.
2. **Remaining `Collection` protocol members** — indexing, subscripting, and
   removal helpers beyond the shared protocol dispatch.
3. **Remaining `Sequence` protocol members** — lazy and contiguous-storage
   operations, plus lexicographic comparison.
4. **`AsyncSequence`** — `for await` over custom types.

To see how these map to GitHub issues, visit the
[issue tracker](https://github.com/siuying/tswift/issues).
